Build your own blog with Jekyll

11 November 2015 • 8 mins read

en | fr
In this tutorial, I'll show you how to build a blog with Jekyll, write our first blog post, add disqus to our post and host it for free on github.

This tutorial is part of a series of tutorials on how to build, maintain and customize a Jekyll blog. If you haven’t read the previous ones, please go through them:

  • Build your own blog with Jekyll
  • A better Jekyll workflow with gulp
  • Customize your Jekyll blog - Part I
  • Customize your Jekyll blog - Part II

Summary

In this tutorial, I’ll show you how to:

  • build a blog with Jekyll,
  • write our first blog post,
  • add disqus to our post,
  • and finally, host it for free on github.

You can view or download the source files on the Jekyll tutorial github repo.

Building a blog with Jekyll

Fist we need to install Jekyll on our computer.

Now that Jekyll is installed, we can generate the blog.

Then go into the generated folder and launch the Jekyll server.

Now you can go to http://127.0.0.1:4000/ and see your blog in action!

Now we want to customize it. All the config informations are in the _config.yml file, located at the root of your folder. Let’s open it in our favorite editor and update it.

We are also going to get rid of the about page because we don’t need it. Simply delete the about.md file in the root of your directory.

Stop and relaunch your jekyll serve command and see the result!

That’s it for the first part! We’ll keep it simple for now and won’t customize the theme of our blog. But stay tuned, I’ll write another tutorial explaining how to do this soon ;)

Writing our first post

First, we need to create the post file in the _posts directory. Post file name needs to follow this convention YEAR-MONTH-DAY-title.MARKUP, so name it like this 2015-11-13-our-first-awesome-post.md.

After this, you can begin to write your first awesome post using the markdown syntax. I won’t go in details on writing markdown as you can find a lot of material explaining this. You could have a look at the github help page for markdown for example.

I’ll just explain how to manage your post images that you want to store locally. You probably already know that to insert an image in your post you’ll use this syntax ![image title](http://image-url). But if you store your images locally, you more likely use a relative url like this one ![image title](/img/image-name.jpg).

So we are going to create an img folder at the root where we’ll store all our images. It will be automatically copied to the _site folder (more on this at the last part of this tutorial).

Now you can add a local image to your post like this ![image title]({{ site.url }}/img/image-name.jpg)

Hosting our blog on github

Now that we know how to add content to our blog, we want to make it live so that we can share our awesome content with others.

Go to your github account and create a new repository.

Then copy the ssh url (or https if you prefer) of your new repository.

Now, open a terminal at the root of your blog project, go to the _site folder and create a new git repository.

Add a remote to your git repository using the url that you copied earlier on github.

Update your site url in _config.yml.

We now need to create the gh-pages branch and then our first commit.

Finally, we push it to github.

You should now be able to see it live at http://username.github.io/blog!

Adding disqus to our post

Now that our post is live, we want to add the possibility for the reader to comment. We’ll use Disqus for that.

First, create an account and then register a new site.

You should then be redirected on the installation page asking you to choose your platform. Choose Universal Code and copy the code of the first step.

Then we need to add this code to our post template file located at _layouts/post.html. Open this file and add at the end of the <article> block the copied code between {% if page.comments %} and {% endif %}.

Now we just have to add comments: true to the front matter of our post and the disqus comment section will load automatically.

Run jekyll build, commit the changes, and push the _site folder to github.

That’s it for this tutorial. Stay tuned for next one.

Bonus: Redirect your custom domain name to your blog

If you own a custom domain name, you’ll probably want to redirect it to your blog. For example, you could redirect blog.yourdomain.com to your github page.

In order to do this, you first need to create a CNAME file at the root of your project. Add one line containing your domain url. So in our case blog.yourdomain.com.

Commit the file and push it to github.
